# Documentation Index

Use this directory as the maintained handoff reference for the deliverable
project. Historical phase notes and closeout experiment logs have been removed
from the shipped documentation surface.

## Start Here

- `active_workflow.md`: active commands for downloading assets and inspecting
  aligned results.
- `active_baseline_handoff.md`: current baseline definition, viewer outputs,
  known limits, and future-work guidance.
- `current_architecture.md`: active modules, package boundaries, and removed
  legacy surfaces.
- `current_milestone_snapshot.md`: compact status snapshot for the current
  handoff point.

## Documentation Rules

- Keep active commands in `README.md`, `active_workflow.md`, and
  `scripts/README.md` aligned.
- Put new architectural decisions in `current_architecture.md` only when they
  reflect maintained behavior.
- Keep one-off measurements out of the deliverable docs unless they become
  maintained validation criteria.
